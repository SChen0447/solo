# 城市记忆档案馆 - 产品需求文档

## 1. 产品概述

### 1.1 产品定位
城市记忆档案馆是一款帮助旅行者系统化整理旅行记忆的全栈应用。它解决了旅行结束后，照片、票根、手写笔记等碎片化纪念品难以系统化整理、且无法与地图位置建立直观关联的问题。

### 1.2 目标用户
热爱旅行、注重旅行体验、希望将旅行记忆系统化保存和展示的用户。

### 1.3 核心价值
- 将照片与地理位置绑定，建立空间维度的记忆关联
- 通过时间线视图，建立时间维度的记忆叙事
- 通过标签和搜索功能，实现记忆的快速检索

---

## 2. 核心功能需求

### 2.1 照片上传与归档

**功能描述**：
用户可以批量上传旅行照片，将每张照片绑定到地图上的具体位置，并添加元数据。

**详细需求**：
- 支持批量选择照片，单张照片大小不超过5MB
- 可手动拖拽标记点到地图上的具体经纬度位置
- 为每个标记点添加：
  - 日期
  - 简短日记（最多200字）
  - 自定义标签（如"美食"、"风景"、"人文"）
- 点击"归档"后，照片以缩略图形式悬浮在地图对应标记点上方

**缩略图样式**：
- 尺寸：150x150px
- 圆角：12px
- 边框：淡灰色
- 阴影：偏移阴影
- 标签：右下角显示标签小圆标

### 2.2 时间线视图

**功能描述**：
以垂直时间河流的形式展示已归档的照片，按日期逆序排列。

**详细需求**：
- 时间河流：半透明淡蓝色弯曲河流线，宽度3px，从页面顶部蜿蜒到底部
- 照片卡片：300x200px，带毛玻璃效果
- 排列方式：沿河流曲线排列
- 交互：
  - 鼠标悬停：卡片放大至1.1倍，显示完整日记内容
  - 点击：在地图上高亮显示对应标记点（呼吸闪烁3次后转为常亮金色）

### 2.3 全局搜索功能

**功能描述**：
通过标签或日期范围过滤照片，地图和时间线同步更新。

**详细需求**：
- 搜索框位置：页面左上方
- 搜索条件：标签或日期范围
- 过滤效果：
  - 匹配项：上浮到顶部，淡黄色光晕环绕
  - 未匹配项：半透明灰色（透明度0.2），下沉到时间河流底部
- 同步：地图和时间线同步过滤显示匹配项

## 3. 界面设计要求

### 3.1 整体风格
- **设计风格**：日式极简风格
- **主色系**：
  - 米白：#FAF0E6
  - 深蓝灰：#2C3E50
- **布局**：
  - 桌面端：地图区域（左60%）| 分隔线 | 时间线（右40%）
  - 分隔线：1px虚线，颜色#BDC3C7

### 3.2 地图区域
- **底图**：Leaflet默认风格冷色调
- **标记点**：圆形渐变图标，直径24px，中心白色到边缘蓝色渐变
- **点击动画**：0.3秒缩放弹性动画（scale: 1 → 1.3 → 1）

### 3.3 时间线区域
- **河流动画**：两侧轻微水波荡漾（SVG滤镜，波纹幅度2px，周期4秒）
- **卡片进入动画**：从底部向上滑入（translateY:20px→0，ease-out，0.5秒）

### 3.4 交互动画
- 所有悬停、点击、搜索过滤的动画时长控制在0.3-0.5秒
- 使用CSS transition实现

### 3.5 响应式适配
- 屏幕宽度 < 768px：
  - 地图和时间线垂直堆叠
  - 地图占60%视高
  - 时间线占40%视高
  - 分隔线变为水平分割线

---

## 4. 性能要求

### 4.1 渲染性能
- 地图上最多同时渲染200个标记点和对应的缩略图卡片
- 在Chrome和Safari上保持60FPS帧率

### 4.2 响应速度
- 搜索过滤响应时间不超过100ms

---

## 5. 技术约束

### 5.1 技术栈
- **前端**：TypeScript + React 18 + Vite 5
- **后端**：Node.js + Express 4
- **地图**：Leaflet
- **HTTP客户端**：axios

### 5.2 约束条件
- 不依赖任何UI组件库
- 不使用状态管理库
- 所有交互逻辑和动画手动实现
- 不引入react-spring、framer-motion等动画库

### 5.3 依赖包
**前端依赖**：
- react@18
- react-dom@18
- typescript
- vite@5
- @types/react
- @types/react-dom
- axios
- leaflet
- @types/leaflet

**后端依赖**：
- express@4
- cors
- multer
- uuid

**开发依赖**：
- concurrently

### 5.4 启动脚本
- 前端：`npm run dev:front`
- 后端：`npm run dev:back`
- 同时启动：`npm run dev`（使用concurrently）
