## 1. 产品概述

创意文化公司虚拟门票系统，为线下展览提供在线艺术化二维码门票生成与核销功能。
- 目标：解决传统门票缺乏艺术感、易伪造的问题，提供美观展示和快速校验的一体化解决方案
- 目标用户：展览参观者（生成/获取门票）、工作人员（核销验证）

## 2. 核心功能

### 2.1 用户角色
| 角色 | 核心权限 |
|------|----------|
| 参观者 | 填写信息生成艺术门票、下载门票PNG、查看门票详情 |
| 工作人员 | 输入/扫描门票ID、验证门票有效性、查看核销结果 |

### 2.2 功能模块
1. **门票生成页面**：用户信息输入、艺术二维码Canvas渲染、门票ID展示、PNG下载
2. **门票核销页面**：门票ID手动输入、二维码图片拖放/粘贴解析、后端验证、结果弹窗展示

### 2.3 页面详情
| 页面名称 | 模块名称 | 功能描述 |
|-----------|-------------|---------------------|
| 生成页面 | 输入面板 | 姓名、展览场次、座位号三个输入框，生成按钮 |
| 生成页面 | 二维码预览区 | 200x200px Canvas艺术二维码渲染、门票ID展示、下载按钮 |
| 核销页面 | 拖放区域 | 虚线边框拖放区，支持粘贴/拖入二维码图片 |
| 核销页面 | 验证面板 | 门票ID输入框、验证按钮、结果弹窗（成功/失败） |

## 3. 核心流程

参观者输入个人信息 → 系统生成唯一门票ID（TKT-XXXX-XXXX格式）→ 基于门票ID+密钥生成加密哈希 → Canvas绘制渐变色彩+圆角模块+中心图案的艺术二维码 → 展示门票ID并提供PNG下载
工作人员输入门票ID或拖入二维码图片 → 前端提取像素矩阵与哈希值 → 发送POST请求至/verify接口 → 后端内存数据库比对并标记状态 → 返回结果并弹窗展示

## 4. 用户界面设计

### 4.1 设计风格
- 主色：#bb86fc（强调色）、#121212（背景）、#1e1e1e（卡片）、#e0e0e0（主文字）
- 按钮：圆角20px，紫色背景白色文字，悬停亮度+20%，0.2s过渡
- 输入框：圆角8px，内阴影，聚焦时box-shadow: 0 0 8px rgba(59,143,252,0.4)
- 字体：现代无衬线字体，标题醒目、正文清晰
- 布局：极简主义卡片式布局，响应式设计

### 4.2 页面设计概述
| 页面名称 | 模块名称 | UI元素 |
|-----------|-------------|-------------|
| 生成页面 | 输入面板 | 三个圆角输入框（姓名/场次/座位）、紫色圆角按钮 |
| 生成页面 | 二维码预览 | 白色圆角卡片、200x200 Canvas、门票ID文本、下载按钮 |
| 核销页面 | 拖放区域 | 深色背景虚线边框、拖入高亮为绿色实线 |
| 核销页面 | 结果弹窗 | 0.3s淡入动画，成功#4caf50+对号，失败#f44336+叉号 |

### 4.3 响应式
- 桌面端（≥768px）：生成页面左右布局4:6（输入区:预览区）
- 移动端（<768px）：单列上下布局
- 所有间距使用rem单位自适应
