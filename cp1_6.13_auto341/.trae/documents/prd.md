## 1. 产品概述

光痕·记忆拼图——一款浏览器内的记忆配对解谜游戏，通过翻转和匹配发光卡片来解锁记忆碎片。玩家在4x4卡片矩阵中寻找配对，匹配成功时绽放彩色粒子与音效，连续匹配触发连击光效，全部完成后播放获胜烟花动画。

- 目标用户：休闲解谜游戏爱好者，追求视觉与音效沉浸感的玩家
- 核心价值：以动态光纹、粒子特效和音频反馈打造高沉浸感的记忆配对体验

## 2. 核心功能

### 2.1 功能模块

1. **游戏主界面**：4x4卡片矩阵、顶部控制栏、背景渐变、粒子特效层
2. **获胜界面**：星型光柱动画、烟花粒子汇聚、得分统计面板

### 2.2 页面详情

| 页面名称 | 模块名称 | 功能描述 |
|---------|---------|---------|
| 游戏主界面 | 卡片矩阵 | 4x4共8对卡片，背面动态光纹，正面手绘几何图案，3D翻转动画 |
| 游戏主界面 | 控制栏 | 显示得分（金色）、已匹配对数（白色）、用时（精确0.1秒）、重置按钮（红色） |
| 游戏主界面 | 粒子系统 | 匹配成功20个彩色光点扩散；失败红色闪烁+缩小复原 |
| 游戏主界面 | 音频系统 | 翻转C4/E4音、成功C5方波+泛音、失败低频噪声脉冲 |
| 游戏主界面 | 连击系统 | 连续成功连击+1，失败重置为0，连击光效与加速计时 |
| 获胜界面 | 星型光柱 | 50个金色光点构成200px光柱，每2秒旋转一圈，持续5秒 |
| 获胜界面 | 烟花动画 | 所有粒子汇聚绽放，显示总分、用时、连击记录 |
| 获胜界面 | 再来一局按钮 | 绿色按钮，点击重新洗牌并重置所有状态 |

## 3. 核心流程

玩家打开游戏 → 看到4x4卡片矩阵（背面朝上） → 点击翻转第一张卡片（C4音） → 点击翻转第二张卡片（E4音） → 判断是否匹配 → 匹配成功：粒子绽放+C5音+得分+10+连击+1 → 匹配失败：红色闪烁+噪声脉冲+连击重置 → 继续翻牌 → 全部匹配完成 → 星型光柱5秒 → 烟花动画 → 显示统计 → 再来一局

## 4. 用户界面设计

### 4.1 设计风格

- **主色调**：深紫灰#1a0f2e → 墨蓝#0d1b2a 径向渐变背景
- **光纹色**：#48dbfb（冰蓝） → #ff9ff3（粉紫）流动渐变
- **强调色**：金色#feca57（得分）、红色#ff6b6b（重置/失败）、绿色#2ecc71（再来一局）
- **字体风格**：无外部字体依赖，使用系统等线/Segoe UI
- **布局风格**：全屏居中，顶部固定玻璃质感控制栏，中央卡片矩阵
- **动效风格**：3D翻转(0.6s ease-in-out)、粒子扩散、光纹流动、星型旋转、烟花绽放

### 4.2 页面设计概览

| 页面名称 | 模块名称 | UI元素 |
|---------|---------|--------|
| 游戏主界面 | 背景 | 径向渐变#1a0f2e→#0d1b2a，全屏Canvas |
| 游戏主界面 | 控制栏 | 半透明玻璃质感(rgba(255,255,255,0.12)+backdrop-filter:blur)，高70px，固定顶部 |
| 游戏主界面 | 卡片 | 120x120px，间距16px，背面光纹动画，正面几何图案 |
| 获胜界面 | 光柱 | 50金色光点，高200px，2秒/圈 |
| 获胜界面 | 统计面板 | 居中弹窗，总分/用时/连击记录 |

### 4.3 响应式设计

- 桌面优先设计，适配320px-1920px视口宽度
- 卡片大小和间距等比缩放（基于视口宽度计算缩放因子）
- Canvas全屏，所有UI元素使用相对定位

### 4.4 性能目标

- 60fps流畅运行
- 翻转动画、粒子系统、音频合成无卡顿
- Canvas 2D渲染，requestAnimationFrame驱动
