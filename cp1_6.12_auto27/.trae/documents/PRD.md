## 1. 产品概述

在线食谱管理与分享平台，让用户能创建、浏览和收藏食谱。面向家庭厨师和美食爱好者，提供直观的食谱管理体验，支持搜索、收藏和创建功能。

## 2. 核心功能

### 2.1 用户角色
| 角色 | 注册方式 | 核心权限 |
|------|----------|----------|
| 普通用户 | 无需注册 | 浏览、搜索、收藏和创建食谱 |

### 2.2 功能模块
1. **食谱列表页**：卡片网格展示所有食谱，搜索过滤，防抖动画
2. **食谱详情页**：完整信息展示，食材勾选，收藏功能
3. **创建食谱页**：表单创建新食谱，动态食材/步骤，拖拽排序

### 2.3 页面详情
| 页面名称 | 模块名称 | 功能描述 |
|----------|----------|----------|
| 食谱列表页 | 搜索栏 | 200ms防抖实时过滤，匹配卡片缩放进入动画，不匹配淡出 |
| 食谱列表页 | 卡片网格 | 每行3列卡片，含封面色块+emoji、名称、作者、评分★☆ |
| 食谱详情页 | 食谱信息 | 封面大图、名称、作者、评分、烹饪时长 |
| 食谱详情页 | 食材列表 | 可勾选表示已准备 |
| 食谱详情页 | 步骤列表 | 有序列表，序号+描述 |
| 食谱详情页 | 收藏按钮 | 心形图标切换，轻提示，localStorage持久化 |
| 创建食谱页 | 基本信息表单 | 名称、作者、评分滑块、烹饪时长 |
| 创建食谱页 | 食材列表 | 动态添加/删除行，每行输入名称和数量 |
| 创建食谱页 | 步骤列表 | 动态添加/删除，拖拽排序 |
| 创建食谱页 | 提交 | Mock接口添加，跳转列表页，新食谱首位滑入动画 |

## 3. 核心流程

用户打开应用后进入食谱列表页，可浏览所有食谱卡片。通过搜索框输入关键词实时过滤食谱。点击卡片进入详情页查看完整信息、勾选食材、收藏食谱。通过导航栏进入创建页填写表单提交新食谱。

## 4. 用户界面设计

### 4.1 设计风格
- 主色：橙黄色 #F4A460，辅色：米白 #FFF8E7 和深棕 #5D4037
- 按钮：渐变背景（#F4A460 → #E67E22），hover亮度提高10%
- 字体：暖色系风格搭配，标题使用粗体衬线体，正文使用无衬线体
- 卡片：圆角16px、浅阴影（0 4px 12px rgba(0,0,0,0.08)），hover阴影加深并放大1.03倍
- 导航栏：固定顶部，半透明毛玻璃效果（backdrop-filter: blur(8px)），含logo、导航链接、收藏角标
- 交互：按钮点击scale(0.96)按压效果，过渡0.2-0.3秒

### 4.2 页面设计概览
| 页面名称 | 模块名称 | UI元素 |
|----------|----------|--------|
| 列表页 | 搜索栏 | 圆角输入框，搜索图标，暖色聚焦边框 |
| 列表页 | 卡片网格 | 3列网格，gap:24px，卡片含色块封面+emoji、标题、作者、评分 |
| 详情页 | 封面大图 | 大尺寸色块+emoji，圆角 |
| 详情页 | 食材列表 | 复选框+食材名，勾选划线 |
| 详情页 | 步骤列表 | 圆形序号+描述文本 |
| 详情页 | 收藏按钮 | 心形图标，填充/空心切换，底部固定 |
| 创建页 | 表单 | 输入框、滑块、动态行列表，渐变提交按钮 |

### 4.3 响应式适配
- 桌面端（≥768px）：卡片3列，完整导航栏
- 平板端（<768px）：卡片2列，汉堡菜单
- 移动端（<480px）：卡片单列，详情页图文垂直排列

### 4.4 动画设计
- 页面切换：从右侧滑入，0.3秒ease-out
- 搜索过滤：匹配卡片缩放进入（0.3秒ease-out），不匹配淡出隐藏
- 新食谱：从顶部滑入动画
- 收藏操作：轻提示3秒自动消失
- 所有过渡：0.2-0.3秒，保持60fps
