## 1. 产品概述

沙漏记忆是一款浏览器端的互动艺术游戏，让用户化身时间旅者，通过操控虚拟沙漏来体验时间流逝与记忆打捞。用户翻转沙漏、调节沙粒流速，观察沙粒堆积成沙丘，同时捕捉夹杂在沙粒中的记忆碎片，将其拼合成完整记忆卡片并解锁新沙漏材质与背景时空布景。

- 核心价值：将时间流逝的抽象概念具象化为沙粒流动的视觉体验，结合记忆碎片收集机制创造沉浸式交互
- 目标用户：追求沉浸式视觉体验和治愈系互动的休闲游戏用户

## 2. 核心功能

### 2.1 功能模块

1. **沙漏交互页面**：沙漏渲染与沙粒动画、翻转操作、速度控制、颜色主题切换、空闲状态检测
2. **记忆碎片收集区**：碎片打捞、卡片合成、收集区展示
3. **解锁与升级系统**：进度条、材质升级、背景布景切换

### 2.2 页面详情

| 页面名称 | 模块名称 | 功能描述 |
|----------|----------|----------|
| 沙漏交互页面 | 沙漏渲染 | SVG绘制铜色描边沙漏，包含上下两层容器和中间细管 |
| 沙漏交互页面 | 沙粒粒子系统 | Canvas渲染2-4px不规则圆角多边形沙粒，颜色随机，带阴影，线性下落+水平偏移 |
| 沙漏交互页面 | 翻转动画 | 点击翻转按钮触发1.5秒180度旋转，缓出函数cubic-bezier(0.4,0,0.2,1)，沙粒反向流动 |
| 沙漏交互页面 | 速度控制 | 拖拽滑块0-100，映射到每秒8-40粒沙粒下落速度 |
| 沙漏交互页面 | 记忆碎片生成 | 每3-5秒随机生成1块25x25px半透明六边形碎片，带金线描边和图标，下落速度比沙粒慢30% |
| 沙漏交互页面 | 碎片打捞 | 点击碎片触发0.3秒缩放放大动画（25→60px）+径向光晕，存入收集区 |
| 沙漏交互页面 | 颜色主题切换 | 5种沙粒颜色主题（沙漠黄、火山灰、海洋蓝、森林绿、极光紫），切换后沙粒和沙丘颜色即时更新 |
| 沙漏交互页面 | 空闲检测 | 连续5秒无操作后沙粒变稀、沙漏半透明、显示时间箴言，再次交互恢复 |
| 记忆收集区 | 碎片存储 | 复古木框区域展示已捞碎片，每3块同类型自动组合 |
| 记忆收集区 | 记忆卡片合成 | 照片碎片→手绘风景灰度图卡片，文字碎片→唐诗宋词节选卡片 |
| 解锁系统 | 进度条 | 300x20px进度条，每合成一张卡片增长10% |
| 解锁系统 | 材质与背景升级 | 铜→银→金三级材质升级，背景从象牙白→星夜深蓝→星云紫 |

## 4. 用户界面设计

### 4.1 设计风格

- **主色调**：沙色(#c2a77a)、铜色(#b87333)、象牙白(#fdf5e6)
- **点缀色**：金色(#ffd700/#d4a017)、深棕(#8b5a2b/#8b4513)
- **按钮风格**：圆角矩形，hover放大1.05倍+8px柔和阴影，点击缩放0.95倍持续0.1秒
- **字体**：Georgia用于箴言文字，系统衬线字体用于标题，无衬线用于UI元素
- **布局风格**：响应式三区布局（沙漏居中/收集区居右），古朴精致风格

### 4.2 页面设计概览

| 页面名称 | 模块名称 | UI元素 |
|----------|----------|--------|
| 沙漏交互页面 | 背景层 | 象牙白→柔和沙色渐变，升级后星夜深蓝/星云紫+星星/银河 |
| 沙漏交互页面 | 沙漏区域 | 500px高SVG沙漏，铜色描边3px，浅沙色填充+3%颗粒纹理 |
| 沙漏交互页面 | 速度滑块 | 0-100滑条，滑块颜色#d4a017 |
| 沙漏交互页面 | 翻转按钮 | 沙漏顶部按钮 |
| 沙漏交互页面 | 颜色主题按钮 | 5个30px圆形色块，2px白色边框，选中时金色光晕 |
| 沙漏交互页面 | 箴言文字 | Georgia 15px，#8b4513，淡入淡出动画 |
| 记忆收集区 | 木框容器 | 400x200px，木纹#8b5a2b，衬纸#f5deb3 |
| 记忆收集区 | 记忆卡片 | 100x80px，轻微投影(2,2,4,rgba(0,0,0,0.3)) |
| 解锁系统 | 进度条 | 300x20px，底色#d2b48c，填充#8b4513→#d4a017渐变 |

### 4.3 响应式设计

- **宽屏(>1024px)**：沙漏居中，记忆收集区居右
- **平板(768-1024px)**：沙漏和收集区上下排列
- **手机(<768px)**：单列布局，沙漏高度缩至300px，按钮和滑块面积增大30%

### 4.4 性能要求

- 主流浏览器(Chrome 110+)保持50+FPS（沙粒<5000时）
- 碎片点击到缩放动画完成响应时间<100ms
- 使用requestAnimationFrame驱动动画循环
- 使用2D Canvas和CSS/SVG渲染，不依赖3D库
